## 强制门

每项都必须 ✅，不允许例外。

- [ ] **WHO** 是一个具体的人，不是人群画像。年龄、角色、地点、你的项目进入她生活的时刻都已命名。
- [ ] **WHY** 描述的痛点独立于你的产品存在。如果你的产品明天消失，这个痛点依然在。
- [ ] **WHY NOW** 有具体触发点（三选一）：(a) 外部变化（技术 / 数据 / 监管 / 行为 / 现有玩家失败），3 年前不成立；(b) 内部积累（能力 / 数据 / 用户洞察到达临界点）；(c) 机会窗口（市场空白 / 竞品空档 / 时间敏感的机会）。
- [ ] **UNDERLYING LOGIC** 解释的是机制，不是结论。如果你拿掉技术机制，方案就不 work — 你能解释为什么。
- [ ] **ANTI-POSITIONING** 至少列了 3 个本项目不是的东西，每个都有理由。

## 质量门

强烈推荐（不强制，但跳过通常意味着 memo 还没准备好）。

- [ ] 整个 memo 装得下**一页打印纸**（≤ ~500 字）。更长，说明你还没决定。
- [ ] WHO 是你**亲眼见过或在咖啡店能认出来的人**。不是市场分层。
- [ ] 每个反定位都能用**一句话** defend。"不是 X 因为我们不解决 Y" — Y 是一个真实的相邻问题。
- [ ] WHY NOW 的触发点能用一句话说清，且可以验证（不是模糊的"时机到了"）。
- [ ] 你**没有**在 positioning 里用以下任何模糊差异化词：
  - "AI 驱动" / "智能" / "聪明"
  - "个性化"（不指定数据源和机制的话）
  - "无缝" / "零摩擦"
  - "革命性" / "下一代"

## 自检问题

每个用 1-2 句回答。答不上来，memo 再过一轮。

1. **如果我在晚宴上跟朋友讲这个项目，30 秒能讲完吗？** 我会怎么说？
2. **这个 positioning 最便宜的验证是什么？**（一个 landing page？5 个用户访谈？concierge MVP？）做过没？
3. **世界要发生什么，才会让这个项目即使做得好也 FAIL？**（这是 "why now" 的反面，更强的测试）
